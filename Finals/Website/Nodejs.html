Server-Side Web Scripting

    Node js

    What is Node js?

    • Node js is a server side platform wrapped around the JavaScript language for building scalable,event-driven applications. Node js utilizes asynchronous programming which implies operations can be executed while the query is being handled. It let them process instead of waiting for one step at a time.

    • Node js files have the enxtension ".js". Just like php, node js can generate a dynamic page content meaning, contents can be changed.

    Creating Server using node js:

    				const http = require('http');
				const hostname = '127.0.0.1';
				const port = 3000;
				const server = http.createServer((req, res) => {
					res.statusCode = 200;
					res.setHeader('Content-Type', 'text/plain');
					res.end('Server is Running! Everything is Fine and working!');
				});
				
				server.listen(port, hostname, () => {
					console.log(`Server running at http://${hostname}:${port}/`);
				});

  • Run it by typing “node server.js” on the command line and go to the browser.

  • Const defines a read-only variable that is not modifiable

  • Server creates a new server to connect the HTTP

  • send(), equals: res.status(code).send(body);

  • res.end will end the process

  • console.log prints the output

    • One other way on Creating Server using node js:

    				var http = require("http");
				http.createServer(function(request, response) {

						response.writeHead(200, {'Content-Type': 'text/plain'});
						response.end('Hello World\n');
				}).listen(8000);

				console.log('Server running at http://127.0.0.1:8000/');

  • Run it by typing “node server.js” on the command line and go to the browser.

  • The var http loads the HTTP module

  • http.createServer creates a server that will listen on port 8000 for requests

  • response.writeHead set the response HTTP header with HTTP status and Content Type

  • response.end sends the response body “hello world”

  • response.end sends the response body “hello world”

  • Write handlers for requests with different HTTP verbs at different URL paths (routes).

  • Integrate with "view" rendering engines in order to generate responses by inserting data into templates.

  • Set common web application settings like the port to use for connecting, and the location of templates that are used for rendering the response.

  • Add additional request processing "middleware" at any point within the request handling pipeline

    • Express Hello World

    				var express = require('express');
				var app = express();

					app.get('/', function(req, res) {		
						res.send('Hello World!');
					});

				app.listen(3000, function() {
					console.log('Example app listening on port 3000!');
				});

  • o use the node js express, you must first type “npm install express”

  • The first two lines require() (import) the express module and create an Express application

  • app, has methods for routing HTTP requests, configuring middleware, rendering HTML views, registering a template engine, and modifying application settings that control how the application behaves

  • app.get() method specifies a callback function that will be invoked whenever there is an HTTP GET request with a path ('/') relative to the site root.

  • Send() returns the string “Hello World”
